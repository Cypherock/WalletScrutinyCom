**Disclaimer**: The authors of this project have contributed to Mycelium.
Andreas Schildbach was so kind and independently tested and
confirmed Mycelium's verifiability
[here](https://github.com/bitcoin-dot-org/bitcoin.org/issues/3221#issuecomment-566466894)
for a past version. Independent tests would be highly welcome!

Here we test if version `3.3.1.1` also can be verified, following the known
procedure using the Mycelium Dockerfile's docker image:

```
$ git clone https://github.com/mycelium-com/wallet-android
$ cd wallet-android
$ git checkout $( git tag | grep 3.3.1.1 | head -n 1 )
$ git clone https://github.com/mycelium-com/wallet-android-modularization-tools.git
$ git submodule update --init --recursive
$ docker run -it --volume $PWD:/mnt --workdir /mnt --rm mycelium-wallet bash
root@83d279fbe702:/mnt# yes | /opt/android-sdk/tools/bin/sdkmanager "build-tools;28.0.3"
root@83d279fbe702:/mnt# ./gradlew clean :mbw:asProdRel
root@83d279fbe702:/mnt# exit
$ apktool d -o fromBuild mbw/build/outputs/apk/prodnet/release/mbw-prodnet-release.apk 
$ apktool d -o fromGoogle path/to/downloaded/mbw-prodnet-release.apk 
$ diff --brief --recursive from* | wc -l
75
```

75 files differ. That is not good and while we know how to fix this, we don't
want to ignore issues, given our involvement.

To verify the build, a non-deterministic behavior in a new library needs to be
dealt with, using disorderfs:

```
$ git clone https://github.com/mycelium-com/wallet-android
$ cd wallet-android
$ git checkout $( git tag | grep 3.3.1.1 | head -n 1 )
$ git clone https://github.com/mycelium-com/wallet-android-modularization-tools.git
$ git submodule update --init --recursive

$ mkdir /tmp/s
$ sudo disorderfs --sort-dirents=yes --reverse-dirents=no --multi-user=yes $PWD /tmp/s
$ cd /tmp/s

$ docker run -it --volume $PWD:/mnt --workdir /mnt --rm mycelium-wallet bash
root@83d279fbe702:/mnt# yes | /opt/android-sdk/tools/bin/sdkmanager "build-tools;28.0.3"
root@83d279fbe702:/mnt# ./gradlew clean :mbw:asProdRel
root@83d279fbe702:/mnt# exit
$ apktool d -o fromBuild mbw/build/outputs/apk/prodnet/release/mbw-prodnet-release.apk 
$ apktool d -o fromGoogle path/to/downloaded/mbw-prodnet-release.apk 
$ diff --brief --recursive from*
Files fromBuild/original/META-INF/CERT.RSA and fromGoogle/original/META-INF/CERT.RSA differ
Files fromBuild/original/META-INF/CERT.SF and fromGoogle/original/META-INF/CERT.SF differ
Files fromBuild/original/META-INF/MANIFEST.MF and fromGoogle/original/META-INF/MANIFEST.MF differ
```

This diff is what we want to see to call it verifiable but ...

The project's build instructions will shortly be updated. In the meantime we
will list the project as **nonverifiable** given here is the only place the
"how to verify" is documented. The provider has to fix the documentation.
