**Disclaimer**: The authors of this project have contributed to Mycelium.
Andreas Schildbach was so kind and independently tested and
confirmed Mycelium's verifiability
[here](https://github.com/bitcoin-dot-org/bitcoin.org/issues/3221#issuecomment-566466894).

Here we test if version `3.2.0.17` also can be verified, following the known
procedure using the Mycelium Dockerfile's docker image:

```
$ git clone https://github.com/mycelium-com/wallet-android
$ cd wallet-android
$ git checkout $( git tag | grep 3.2.0.17 | head -n 1 )
$ git clone https://github.com/mycelium-com/wallet-android-modularization-tools.git
$ git submodule update --init --recursive
$ docker run -it --volume $PWD:/mnt --workdir /mnt --rm mycelium-wallet bash
root@83d279fbe702:/mnt# yes | /opt/android-sdk/tools/bin/sdkmanager "build-tools;28.0.3"
root@83d279fbe702:/mnt# ./gradlew clean :mbw:asProdRel
root@83d279fbe702:/mnt# exit
$ apktool d -o fromBuild mbw/build/outputs/apk/prodnet/release/mbw-prodnet-release.apk 
$ apktool d -o fromGoogle path/to/downloaded/mbw-prodnet-release.apk 
$ diff --brief --recursive from*
Files fromBuild/original/META-INF/CERT.RSA and fromGoogle/original/META-INF/CERT.RSA differ
```

The built APK and the downloaded APK only differ in files concerning the
signature. Our verdict: This wallet is **verifiable**.
